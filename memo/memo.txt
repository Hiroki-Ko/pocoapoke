■ClaudeFlare周りについて
◎ClaudeFlare Workers
　ClaudeFlareが用意してくれるサーバの代わり。
　自分でレンタルサーバー借りなくても、Cloudflare 上でプログラム（API）が動かせる。
　他ぼサーバ提供サービスとの違いとして、無料枠が大きく個人開発レベルならタダで動かせる
◎ClaudeFlare Pages
　静的サイト+APIをまとめて管理できるサービス
　※普通なら別々で管理して連携しないといけないところをまとめることができる(ドメインも同じ)
　特徴としてfunction配下に置いたコードをAPIとして動かすことができる(=中身はWorkersと同じエンジンで動く)
　ただしwindowsでのローカル開発では不安定
◎Vite
　ClaudeFlareとは無関係の「フロントエンド専用のローカル開発サーバー」(開発ツール)
　Pagesがローカル環境ではうまく動かせないので代わりに使う
◎Miniflare
　Workers をローカルで動かすためのエンジン
　Pagesのfunctionsでやりたいことを、ローカルではMiniflare(= Workers dev)で代用する

ローカル：
  フロント → Vite
  API → Workers dev（Miniflare）

本番：
  Pages にまとめてデプロイ

import { getAssetFromKV } from "@cloudflare/kv-asset-handler";

export default {
  async fetch(request, env, ctx) {
    const url = new URL(request.url);

    // /test は API として処理
    if (url.pathname === "/test") {
      await env.POKEMON.put("test", "hello world");
      const value = await env.POKEMON.get("test");
      return new Response(value);
    }

    // 静的ファイル（React）を返す
    try {
      return await getAssetFromKV(
        { request, waitUntil: ctx.waitUntil.bind(ctx) },
        { ASSET_NAMESPACE: env.ASSETS }
      );
    } catch (e) {
      return new Response("Not found", { status: 404 });
    }
  }
}

【プロジェクト概要】
- Cloudflare でゲーム進捗管理アプリを作成中
- フロント：React + Vite
- バックエンド：Cloudflare Functions（Workers）
- DB：D1

【問題】
- Windows で Pages dev が壊れており、静的ファイルと Functions が正しく動かない
- そのためローカルで API が動かず、フロントから fetch が失敗していた

【理解したこと】
- Workers = Cloudflare のサーバー
- Pages = 静的サイト + API をまとめて公開できるサービス
- Pages Functions の中身は Workers と同じ
- Vite = フロントをローカルで動かすツール
- Miniflare = Workers をローカルで動かすエンジン
- wrangler dev = Miniflare を使って Workers をローカルで動かすコマンド

【ローカル開発の最適解】
- フロント：Vite（5173）
- API：Workers dev（Miniflare）（8787）
- Pages dev は使わない（Windows で壊れるため）

【次にやるべきこと】
1. プロジェクトルートに wrangler.toml を置く
2. Workers 用に設定する
3. `npx wrangler dev` で API を起動
4. `npm run dev` でフロントを起動
5. フロントの API URL を 8787 に向ける
